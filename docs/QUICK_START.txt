Q: How do I run both servers?
A: Open 2 terminals. One for "cd backend && npm run dev"
   and another for "cd frontend && npm start"

Q: What port does each service use?
A: Backend: 5000, Frontend: 3000, Docker: standard ports

Q: How do I test integrations?
A: Upgrade to Starter tier, then select integrations in build

Q: Can I self-host?
A: Yes! Use Docker Compose or deploy to any cloud platform

Q: How do I deploy to production?
A: Use docker-compose up or deploy backend/frontend separately

Q: Where are API keys stored?
A: Hashed with SHA-256 in backend (production: DB)

Q: Can I modify the integrations?
A: Yes! Edit backend/lib/integrations.js and server.js

Q: What's the build lock for?
A: Prevents concurrent builds per user - race condition safety

Q: How do I add new integrations?
A: 1. Add to integrations.js
   2. Add code generator to integrationHandler.js
   3. Add tier check in server.js
